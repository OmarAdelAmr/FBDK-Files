This page documents changes since the 2004-11-19 release of the FBDK.

FBDK [FBRT] [MVCD]

  • The FBDK has been tested to work with the J2SE 5.0 SDK, which has a more "modern" look and feel. The FBDK may also work with the J2SE 1.4.2 SDK.
  • Customization procedures have been simplified.
  • Graphical editing of network segments and links is now possible in system configuration diagrams.
  • The new style of representing parameters as parenthesized assignment lists is now supported.
  • Graphical editing of service sequences is now supported.
  • XML Parsing Only
    • Due to increasing complexities in the source format introduced by the new features described above, only parsing of XML-formatted source text is now supported.
    • Reading of *.src files is no longer supported.
    • Parsing of text from the Source view is no longer supported.

FBRT [FBDK] [MVCD]

  • The FBRT has been tested to work with the J2SE 1.4.2 SDK and may not work with earlier versions of the SDK.
  • HMI classes formerly located in the fb.rt.hmi package have been converted to the Java 2 Swing GUI and are now located in the fb.rt.swing package; however, IEC 61499 source code for these classes is still located in the fbdk/src/hmi folder.
  • Internal implementation of the abstract superclass (fb.rt.FBInstance) of all FB types has been changed; you should re-generate and recompile all library elements you have previously created (use the  File=>SaveAs=>Java menu item).
  • Internal implementation of the fb.rt.net classes has been changed to provide "pluggable" presentation protocols. This will not affect you unless you have subclassed these communication service interfaces.

MVCD [FBDK] [FBRT]

  • Substantial improvements and additions have been made to the MVCD framework.
  • Process control and sortation/inspection applications have been incorporated into the framework and a new pallet handling example has been added.
