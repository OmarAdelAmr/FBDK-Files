What's New

This page documents changes since the 20061017 release of the FBDK.

FBDK [FBRT] [MVCD]

Features

  • Editor support for the Tagged Data design pattern has been added.
  • Yellow highlighting of selected graphic elements has been added.
  • Documentation comments are now generated for many of the fields and methods in Java™ files created with the Save As dialog.
  • Launching of system configurations has been streamlined.
  • Documentation of system configurations for configurability testing has been linked to Clause 7 of the Compliance Profile.
  • Java™ documentation has been provided for some core runtime classes with a new Javadoc™ entry on the Help menu.
  • Documentation of the MATRIX data type and associated function blocks is provided as part of the Java documentation.

Bug fixes

  • Interfaces of the PACK_BYTE, UNPK_BYTE, FB_ROL_DWORD, FB_ROL_LWORD, FB_ROL_WORD, FB_ROR_BYTE, FB_ROR_DWORD, FB_ROR_LWORD and FB_ROR_WORD function block types have been corrected in the src/math directory (thanks to Ingo Hegny at ACIN for spotting these).
  • FLASHER4.fbt and associated files (FLASHER_TEST.sys, etc) have been moved from src/ita to src/doc, since these have more to do with tutorial documentation than with the Compliance Profile.

FBRT [FBDK] [MVCD]

Features

  • New TAGIN and TAGOUT function block types have been added to the fb.rt.net package to support the Tagged Data design pattern.
  • The hmi package has a new SPINNER type for INT input and a new MOMENTARY pushbutton type.
  • The DELETE command for connections and FB instances defined in Clause 6 of the Compliance Profile has been fully implemented.

Bug fixes

  • Corrected implementation of FB_ROL_DWORD, FB_ROR_DWORD.
  • Previous verions of the FBRT accepted syntactically incorrect parameter values for ARRAYs of STRING or WSTRING, e.g., ["a",b,c,"d"] was accepted as a valid parameter value for a variable of type WSTRING[4]. Only correct syntax is now accepted, e.g., the previous example must be corrected to ["a","b","c","d"] or an INVALID_OBJECT error will be flagged.

Other Changes

  • FLASHER4.class has been moved from fb.rt.ita to fb.rt.doc.

MVCD [FBDK] [FBRT]

Features

  • A new Tagged Data design pattern has been added.
  • The MVCD methodology and examples have been updated to incorporate the Tagged Data design pattern.

Bug fixes

  • MECH2_MDL has been updated to report correctly its sensor states on the first CLK cycle.
