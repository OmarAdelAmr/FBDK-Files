Build and Test an Adapter Interface Type

  1. Start the FBEditor application by running the editor.bat file.
  2. Create a new adapter interface type in the editor by:
    • Opening a file containing an existing adapter interface type, for instance the MECH_ADP.adp file in the src\mva folder, or
    • Clicking the File => New => AdapterType menu item. Here is how the screen will look if you do this:
  3. Edit the adapter's elements:
    • Edit the adapter's properties.
    • Edit the adapter interface.
    • Edit the adapter's service sequences to document the expected socket/plug interactions.
  4. When your editing is complete, save your work with the File=>Save As=>XML... menu item.
    • It is a good idea to save your work from time to time while editing, too!
  5. Generate and compile a JavaTM source file with the File=>Save As=>Java... menu item.
  6. Adapters cannot be tested independently of their use as plugs and sockets, so you will have to create special composite function block types and a system configuration to do this testing. For an example see the TEST_ADP_SYS configuration.
