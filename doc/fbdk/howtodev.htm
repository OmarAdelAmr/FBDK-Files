Build and Test a Device Type

  1. Start the FBEditor application by running the fbeditor.bat file.
  2. Create a new device type in the editor by:
    • Opening a file containing an existing device type, for instance the RMT_DEV.rev file in the src\ita folder or the TEST_DEV2.dev file in the src/test folder; or
    • Clicking the File => New => DeviceType menu item. Here is how the screen will look if you do this:
  3. Edit the device type's elements:
    • Edit the device type's properties.
    • Edit the device type's interface (data inputs, i.e., parameters, only).
    • Edit the device type's body by clicking on the resource type name in the Navigation area and editing the graphic layout of the FBD in the Worksheet/Viewport area.
    • Edit the resource types and FB types supported by the device.
  4. When your editing is complete, save your work with the File=>Save As=>XML... menu item.
    • It is a good idea to save your work from time to time while editing, too!
  5. Generate and compile a Java source file with the File Save As... menu item.
  6. Device types cannot be tested independently of their system configuration, so you will have to create a system configuration to do this testing. For an example see the TEST_SYS3 configuration.
