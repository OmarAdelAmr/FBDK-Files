What's New

This page documents changes since the 20060720 release of the FBDK.

FBDK [FBRT] [MVCD]

Features

  • A Run button has been added to the menu bar to facilitate this commonly used operation.
  • A new template has been added for simple function block types, which have only a REQ/CNF event pair and a single algorithm to be executed.
  • The functional programming paradigm has been adopted for efficient runtime execution of algorithms in the IEC 61131-3 FBD language:
    • Algorithms in the FBD language are restricted to use only "pure" IEC 61131-3 functions, which have a single output and no internal state.
    • A new file type "*.fnt" and icon are added for function templates, which are used for generation of code corresponding to IEC 61131-3 functions in FBD algorithms.
    • "Stateful" IEC 61131-3 function block types (CTU, etc.) and functions with multiple outputs (DIV_INT, etc.) have been removed from the FBDK library.
    • IEC 61499 equivalents may be substituted for the removed types, e.g., E_CTU for CTU and FB_DIV_INT for DIV_INT.
    • Basic function blocks with FBD algorithms using the removed function block types may not load and will not compile correctly, and may have to be edited in raw XML form to remove the obsolete function block types.
  • Comparison, selection and string function blocks have been given their own folders (compare, select and strings, respectively).
  • Items have been added to the Help menu for the FBDK Discussion Group and this download area.

Bug fixes

  • No significant bug fixes in this release.

FBRT [FBDK] [MVCD]

  • Real-time performance of output event dispatching has been improved.
  • Real-time performance of "Simple" function blocks (FB_AND, etc) has been improved.
  • Real-time performance of algorithms written in the IEC 61131-3 FBD language has been significantly improved with the functional programming paradigm.
  • Real-time performance of algorithms written in the IEC 61131-3 LD language has been improved through use of the Java™ conditional Boolean operators && and || in generated code.
  • Real-time performance and memory requirements of many event-handling blocks have been improved.

MVCD [FBDK] [FBRT]

  • Functional allocation is refactored in the Pick and Place example and the control has been re-implemented as a simple FB.
