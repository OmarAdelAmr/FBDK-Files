What's New

This page documents changes since the 20060206 release of the FBDK.

FBDK [FBRT] [MVCD]

Features

  • Launching is again done via the fbeditor.bat file to allow for differences in JDK™ implementations.
  • XML/src selection has been changed from radio buttons to tabs, and the Parse/Revert functions have been moved to the File menu.
  • Shortcut buttons have been added to the menu bar for the most often used File menu items.
  • Move Up/Down buttons have been added to the menu bar.
  • A Close current tab button has been added at the right of the menu bar.
  • File Save As functions are consolidated into a single File Save dialog box, with all file locations determined by package name.
  • Automatic file backup is added to the File Save function.
  • A popup menu is now provided for adding variables to structured data types.
  • A file chooser has been added to dialogs for editing FB, device and resource instances and variables, to simplify the adding of new types.
  • When doing System configurations, it is now possible to add and edit FB instances and connections in Resources which come from Device types. This feature was not previously supported; FBNetwork elements could only be added and edited in Resources which were added to the Device instances as part of the System configuration.
  • The path parameter is no longer used in the editor.ini file; both source and object code hierarchies are derived from the src directory hierarchy.
  • Documentation has been updated, reorganized and expanded.

Bug fixes

  • A bug in parsing of FBNetworks in Device instances was found and fixed.
  • FBNetworks contained in Resources inside Device Types are now correctly generated by the File Save as Java operation.
  • The upper value of UDINT variables was incorrectly set to 231-1; it is now correctly set to 232-1 and an ADD_UDINT is provided in src/plc/math for checking this.
  • Drag and drop connection to an input variable which is already connected to a parameter is now recognized as an error and disallowed.
  • A number of errors in generation of Java code for Device and Resource types have been corrected.

FBRT [FBDK] [MVCD]

  • A new batch file is provided to enable launching of FB instances and system configurations independent of the FBEditor application.
  • Many changes have been made to improve runtime efficiency and management. This requires regeneration and recompilation of any Java code you may have previously generated for function block types, data types, adapter types, resource types or device types.
  • Documentation has been updated, reorganized and expanded.

MVCD [FBDK] [FBRT]

  • A new Pick and Place example has been added to the MVCD Home Page. Interesting new features include new function blocks for mode control (MODE_AMS) and Boolean input with user-defined graphics (IMG_BOOL), and a first example of mechatronic design.
  • A new (and as yet incomplete) HVAC (Heating, Ventilating, Air Conditioning) example has been added to the MVCD Home Page. An interesting feature is the use of least-squares linear regression to smooth noisy input data.
  • See above description of FBRT changes.
  • Documentation has been updated, reorganized and expanded.
